# Context Example: Creative Studio Surgery

## Target

An implemented browser color-grading studio. Editors work for hours with a frame preview, timeline, scopes, layers, inspector, keyboard commands, autosave, undo, and export.

## Context Read

`studio/editor + create + current frame/timeline + expert repeated use + multi-panel workspace + selected/unsaved/export proof`

## Wrong Diagnosis

Calling it a dashboard because it has panels and metrics leads to KPI cards, summary-first hierarchy, decorative charts, and a reduced preview. That repair optimizes observation while the user is trying to manipulate an artifact.

## Preserve

- Keyboard grading and tool switching.
- Current clip, time, selection, zoom, and layer scope.
- Undo history, autosave, conflict handling, and export settings.
- Accurate scopes and comparison mode.

## Systemic Cut Order

1. Give preview and timeline sovereignty; demote GPU/render diagnostics.
2. Stabilize tool mode, selection scope, save state, and undo visibility.
3. Regroup inspector controls by color intent and selected object.
4. Assign panel scroll ownership; remove nested scroll and width jumps.
5. Normalize control rows, icon source, optical centering, gaps, and focus states.
6. Derive expression from image material, scopes, handles, or direct manipulation.

## Proof Matrix

- States: blank project, loaded timeline, selected/multi-selected layer, unsaved/saving/saved, conflict, export failure.
- Interactions: keyboard tool switch, undo/redo, panel collapse, zoom/pan, parameter edit, comparison toggle.
- Viewports: primary desktop workspace and supported minimum width.
- Detail evidence: inspector rows, toolbar icons, nested scroll boundaries, save status, and focus at DPR 2.

## Claim Limit

This calibration shows how context changes diagnosis. It proves nothing about a real studio until its source, runtime states, shortcuts, persistence, and captures are inspected.
